Perlin Noise

Perlin noise is a gradient type noise developed by Ken Perlin and presented in a SIGGRAPH paper in 1985. This is often used to generate realistic textures and terrain due to its natural-esque gradient.


An organic surface generated with Perlin noise


Perlin noise is generally a function of 3 different parameters which are usually co-oridnates of a point. This is very often used on a 2D grid or a cubical space.


Each grid point is associated with a pseudo random gradient vector. To find the noise at any point on the grid, the first step is to find the cell inside which the point lies. For each point on that cell, an offset vector is calculated and then the dot product is obtained between the gradient vector and the offset vector. The final step is to interpolate between each of the obtained values for every corner.


Perlin noise can also be controlled to implement better looking textures and smoother noise by using octaves.

  • Octaves - The number of times the noise value is added
  • Persistence - The amount of influence each successive octave has
  • lacunarity - The scale of noise each successive octave uses
  • frequency - The scale of noise

Marching Cubes

The marching cubes algorithm works on a dommain whose each co-ordinate is associated with a value. These values are then used to determine the surface of the mesh to be generated. The threshold on the value is called the isoLevel and the surface generated is thus called iso surface.

Marching cubes demonstration

THe algorithm works on the fact that there are only 15 unique possible cases that can occur during the generation. The other cases are simply a symmetrical variation of one of these 15 cases. There are a total of 2^8 = 256 cases which are mapped to one of these 15 cases.
